This is not a complete list of all commands, instead if focuses mostly on commands that are needed for GPRS/3G/UMTS/HSDPA and a few other more “exotic” commands that aren’t too well known. I wrote this mainly for myself, but I figure other might find it useful as well.<br><br>### PIN<br><br>**Command**: AT+CPIN? <br>**Response**: +CPIN: <code><br>**Description**: Check PIN status.<br><br><code> can have several values, these are the more common ones<br><br>READY<br>Not waiting for PIN (no PIN or PIN already entered)<br>SIM PIN<br>Waiting for SIM PIN code.<br>SIM PUK<br>Waiting for SIM PUK code.<br><br>**Command**: AT+CPIN=<pin>\[,<newpin>\]<br>**Response**: OK \| +CME ERROR: <error><br>**Description**: Enter/change PIN.<br><br>**Example**<br><br>> \>AT+CPIN?<br>> +CPIN: SIM PIN<br>> OK<br>> <br>> \>AT+CPIN=”1234″<br>> OK<br>> <br>> \>AT+CPIN?<br>> +CPIN: READY<br>> OK<br><br>### Network registration<br><br>**Command**: AT+CREG?, AT+CREG=? , AT+CREG=<n><br>**Response**: +CREG: <n>,<stat>,\[,<lac>,<ci>\[,<AcT>\]\] \| +CME ERROR<br>**Description**: Get network registration status and controls unsolicited status callback, which when turned on will send a +CREG message with the new network status when something changes.<br><br><n><br><br>0. Disable unsolicited status callback.<br><br>* Enable unsolicited status callback, +CREG: <stat><br>* Enable unsolicited status callback, +CREG: <stat>,\[,<lac>,<ci>\[,<AcT>\]\]<br><br><stat><br><br>0. Not registered, not searching<br>1. Registered to home network<br>2. Not registered, searching for network<br>3. Registration denied<br>4. Registered, roaming<br><br><lac> Location area code<br><ci> UTRAN/GERAN cell ID<br><br><AcT> Network access type<br><br>0. GSM<br>1. Compact GSM<br>2. UTRAN<br>3. GSM with EGPRS<br>4. UTRAN with HSDPA<br>5. UTRAN with HSUPA<br>6. UTRAN with HSDPA and HSUPA<br><br>UTRAN is a short for UMTS Terrestrial Radio Access Network<br><br>The following example shows that full unsolicited status callback is enabled, that the device is registered to its home network followed by an location area code followed by the UTRAN cell ID. Unfortunately this device does not show access type with this command.<br><br>**Example:**<br><br>> \>AT+CREG=2<br>> OK<br>> \>AT+CREG?<br>> +CREG: 2,1,”048A”,”58B2″<br><br>**Command**: AT+COPS?,<br>**Response**: +COPS: (<mode>,\[<format>,<oper>\[,<AcT>\]\]),…, (<modeN>,\[<formatN>,<operN>\[,<AcTN>\]\])<br><br>**Command**: AT+COPS=?<br>**Response**: +COPS: <stat>, long <oper>, short <oper>, numeric <oper>, <AcT><br><br>**Response**: +CME ERROR:<br>**Command**: AT+COPS=<mode>,\[<format>,<oper>\[,<AcT>\]\]<br>**Response**: OK \| +CME ERROR<br><br>**Description**: Get/set current GSM/UMTS network operator, list available operators. This can be used to change for example access type and switch network.<br><br><mode><br><br>0. Automatic network selection (<oper> ignored)<br>1. Manual network selection, <oper> must be present, <AcT> is optional.<br>2. Deregister from network.<br>3. Set <format only, no registration/deregistration.<br>4. Manual selection with automatic fall back (enters automatic mode if manual selection fails).<br><br><format><br><br>0. Long alphanumeric string<br>1. Short alphanumeric string<br>2. Numeric ID<br><br><oper><br>String (based on <format>) that identifies the operator.<br><br><stat><br><br>0. Unknown<br>1. Available<br>2. Current<br>3. Forbidden<br><br><AcT> Network access type<br><br>0. GSM<br>1. Compact GSM<br>2. UTRAN<br>3. GSM with EGPRS<br>4. UTRAN with HSDPA<br>5. UTRAN with HSUPA<br>6. UTRAN with HSDPA and HSUPA<br><br>The following example shows that the devices is connected to the operator called “3″ using UTRAN (which is UMTS, also called 3G). Listing available networks shows the current network, an additional UTMS network called Sweden3G that is forbidden and a GSM network called “3″ that is available.<br><br>**Example:**<br><br>> \>AT+COPS?<br>> +COPS: 0,0,”3″,2<br>> <br>> AT+COPS=?<br>> +COPS: (2,”3″,”3″,”24004″,2),(1,”3″,”3″,”24008″,0),(3,”Sweden 3G”,”Sweden3G”,”2)<br><br>**Command**: AT+COPN<br>**Response**: +COPN: <numeric1>,<alpha1>, …, +COPN: <numericN>,<alphaN><br>**Description**: Read operator names stored in device memory.<br><br>**Example:**<br><br>> +COPN: “20408″,”NL KPN”<br>> +COPN: “20412″,”NL Telfort”<br>> +COPN: “20416″,”T-Mobile NL”<br>> +COPN: “20420″,”Orange NL”<br>> …<br>> +COPN: “24002″,”3 SE”<br>> +COPN: “24004″,”SWEDEN”<br>> +COPN: “24005″,”Sweden 3G”<br>> +COPN: “24008″,”Telenor SE”<br>> +COPN: “24010″,”S COMVIQ”<br><br>### Define PDP (Packet Data Protocol) context<br><br>**Command**: AT+CGDCONT=<cid> \[,<pdptype> \[,<apn>\[,<pdpaddr> \[,<dcomp>\[,<hcomp\]\]\]\]\]<br>**Response**: OK \| ERROR<br><br>**Command**: AT+CGDCONT?<br>**Response**: +CGDCONT: <cid1>,<pdptype1>,<apn1>,<pdpaddr1><dcomp1>,<hcomp1>, …, <cidN>,<pdptypeN>,<apnN>,<pdpaddrN><dcompN><br><br>**Command**: AT+CGDCONT=?<br>**Response**: +CGDCONT: (<cid\_range>),<pdptype>,,,(<dcomp\_range>),(<hcomp\_range>)<br>**Description**: Allows configuration of one or several packet data protocol context which forms the base of a data connection.<br><br><cid> PDP context ID, minimum value is 1, maximum value depends on device and can be found with the =? command.<br><br><pdptype> String parameter identifying the protocol type<br>IP – Internet Protocol<br>IPV6 – Internet Protocol, version 6<br>PPP – Point to Point Protocol<br><br><apn> String that identifies the Access Point Name in the packet data network.<br><br><pdpaddr> Requested address, if null (0.0.0.0) an address is requested dynamically.<br><br><dcomp> PDP data compression control, off by default.<br><br><hcomp> PDP header compression control, off by default.<br><br>**Example**:<br><br>> \> AT+CGDCONT=1,”IP”,”bredband.tre.se”<br>> OK<br>> <br>> \>AT+CGDCONT?<br>> +CGDCONT: 1,”IP”,”bredband.tre.se”,”0.0.0.0″,0,0<br>> <br>> \>AT+CGDCONT=?<br>> +CGDCONT: (1-16),”IP”,,,(0-2),(0-4)<br>> +CGDCONT: (1-16),”PPP”,,,(0-2),(0-4)<br>> +CGDCONT: (1-16),”IPV6″,,,(0-2),(0-4)<br><br>### PDP Authentication<br><br>**Command**: AT$QCPDPP=<cid>,<auth\_type>,<auth\_name>,<auth\_pwd><br>**Response**: OK \| ERROR<br>**Description**: A Qualcomm command, defines authentication parameters for the PDP context id <cid>.<br><br><auth\_type><br><br>0. None<br>1. PAP<br>2. CHAP<br><br><auth\_name> and <auth\_pwd> are strings with the authentication information.<br><br>### Signal quality<br><br>**Command**: AT+CSQ<br>**Response**: +CSQ: <rssi>,<ber><br>**Description**: Returns signal quality.<br><br><rssi> Received Signal Strength Indicator<br><br>0<br>\-113 dBm or less<br>1<br>\-111 dBm<br>2 to 30<br>\-109 to -53 dBm<br>31<br>\-51 dBm or greater<br>99<br>not known or not detectable<br><br><ber> Bit Error Rate, in percent 0..7.<br>99 not known or not detectable<br><br>A note on the RSSI (received signal strength), dBm is a decibel (logarithmic) scale with a reference of 1 milliwatt thus 0 dBm equals a received signal of 1 mW.<br>Signal strength is usually lower than 1 mW and therefore below 0, so the larger (closer to 0) the better signal strength.<br>You can convert the RSSI to dBM with<br>![[Resources + Media/attachments/3G_modem_commands.resources/unknown_filename.3.png]]<br><br>**Example**<br><br>> \>AT+CSQ=?<br>> +CSQ: (0-31,99),(99)<br>> OK<br>> <br>> \>AT+CSQ<br>> +CSQ: 14,99<br><br>### Manufacturer/model/revision identification<br><br>**Command**: AT+GMI<br>**Response**: Manufacturer Identification string<br><br>**Command**: AT+GMM<br>**Response**: Model Identification string<br><br>**Command**: AT+GMR<br>**Response**: Revision Identification string<br><br>**Example**:<br><br>> \>AT+GMI<br>> Option N.V.<br>> OK<br>> <br>> \>AT+GMM<br>> GTM378<br>> OK<br>> <br>> \>AT+GMR<br>> 2.4.6Hd (Date: Oct 04 2007, Time: 11:59:08)<br><br>### Option N.V. proprietary AT-style commands<br><br>**Command**: AT\_OWANCALL=<pdp context>,<enabled>\[,<callback enabled>\]<br>**Response**: OK<br>**Description**: Used to initiate UMTS/HSDPA connections.<br><br><pdp context> – Existing, valid, PDP context that specifies the intended APN to connect to.<br><enabled> – 1 = Enable connection, 0 = Disable connection (disconnect)<br><callback enabled> – 1 = Asynchronous callback when connection is established, 0 = silent<br><br>**Example**:<br><br>> \>AT\_OWANCALL=1,1,1<br>> OK<br>> \_OWANCALL: 1, 1<br>> <br>> \>AT\_OWANCALL=1,0<br>> OK<br>> \_OWANCALL: 1, 0<br><br>**Command**: AT\_OWANDATA=<pdp context><br>**Response**: \_OWANDATA: <pdp context>, <ip address>, <route?>, <nameserver 1>, <nameserver 2>, <unknown>, <unknown>, <speed><br>**Description**: Retrieve IP configuration from an established connection previously created with AT\_OWANCALL<br><br>**Example**:<br><br>> \>AT\_OWANDATA=1<br>> \_OWANDATA: 1, 79.138.181.171, 0.0.0.0, 80.251.192.244, 80.251.192.245, 0.0.0.0, 0.0.0.0, 72000<br><br>### References<br><br>[3GPP TS 27.007 V8.3.0 (2008-03) – AT command set for User Equipment
